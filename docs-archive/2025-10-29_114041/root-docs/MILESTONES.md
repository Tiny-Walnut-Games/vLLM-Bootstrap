## How to Use Milestones

### For Maintainers:

1. **Create milestones in GitHub** using the definitions above
2. **Assign issues to milestones** as they're created or triaged
3. **Track progress** using GitHub's milestone view
4. **Update milestone descriptions** with links to key issues
5. **Close milestones** when all exit criteria are met

### For Contributors:

1. **Check current milestone** to see what's being worked on
2. **Filter issues by milestone** to find relevant work
3. **Suggest milestone assignments** for issues you create
4. **Track progress** toward the next release

---

## Milestone Labels

Use these labels in combination with milestones:

- `milestone:blocker` - Must be resolved before milestone completion
- `milestone:nice-to-have` - Would be great but not required
- `milestone:stretch-goal` - If time permits

---

## Milestone Review Schedule

- **Weekly:** Review progress on current milestone
- **Monthly:** Adjust milestone scope based on progress
- **Quarterly:** Review upcoming milestones and roadmap
