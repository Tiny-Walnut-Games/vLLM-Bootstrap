# Setup Verification Guide

After running `initial-bootstrap.sh`, use these scripts to verify that your installation was successful and all components are in place.

---

## 🎯 Quick Start

### For Linux/WSL Users:
```bash
cd ~/.config/llm-doctrine
chmod +x scripts/verify-setup.sh
./scripts/verify-setup.sh
```

### For Windows Users (PowerShell):
```powershell
cd C:\Users\YOUR_USERNAME\RiderProjects\vLLM-Bootstrap
.\scripts\verify-setup.ps1
```

---

## ✅ What Gets Checked

### 1. **Python Environment**
- ✅ Virtual environment exists at `~/torch-env`
- ✅ PyTorch is installed
- ✅ vLLM is installed

### 2. **Configuration Files**
- ✅ `models.conf` exists and is readable
- ✅ `ports.conf` exists and is readable

### 3. **Helper Scripts** (Generated by `initial-bootstrap.sh`)
- ✅ `daily-bootstrap.sh` exists and is executable
- ✅ `test-connection.sh` exists and is executable
- Script permissions are fixed automatically if needed

### 4. **Authentication**
- ⚠️ HuggingFace token configuration (warning if not present)

### 5. **GPU/CUDA** (Optional)
- ⚠️ NVIDIA GPU detection (warning if not found, but CPU mode still works)

### 6. **Repository Structure**
- ✅ All expected directories and files are present
- ✅ Documentation files exist

---

## 📋 Exit Codes

```
0 = ✅ All checks passed - ready to use
1 = ❌ One or more checks failed - review errors
```

---

## 🔍 Verbose Mode

For detailed diagnostic output:

### Linux/WSL:
```bash
./scripts/verify-setup.sh --verbose
```

### PowerShell:
```powershell
.\scripts\verify-setup.ps1 -Verbose
```

This will show:
- Specific versions of installed packages
- GPU information if available
- Detailed error messages

---

## 🚀 What To Do After Verification

### If All Checks Pass ✅

You're ready to use vLLM-Bootstrap!

```bash
# 1. Activate Python environment
source ~/torch-env/bin/activate

# 2. Navigate to installation directory
cd ~/.config/llm-doctrine

# 3. Launch a model (example: QA tier)
./daily-bootstrap.sh qa

# 4. In a new terminal, test the connection
./test-connection.sh 8500

# 5. Chat with the model via curl
curl http://localhost:8500/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mistralai/Mistral-7B-Instruct-v0.3",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

---

### If Some Checks Fail ❌

**Common issues and solutions:**

#### Missing Python Environment
```bash
# Re-run the bootstrap script
./scripts/initial-bootstrap.sh
```

#### Scripts Not Executable
```bash
# Fix permissions
chmod +x ~/.config/llm-doctrine/*.sh
chmod +x ~/.config/llm-doctrine/scripts/*.sh
```

#### HuggingFace Token Not Found
```bash
# Log in manually (optional, needed for some models)
source ~/torch-env/bin/activate
huggingface-cli login
```

#### GPU Not Detected
- This is a warning, not a failure - CPU mode will still work
- vLLM will automatically use CPU if no GPU is available
- Check GPU drivers: `nvidia-smi`

#### Other Failures
1. Check the error message from the script
2. Review the [Troubleshooting Guide](../wiki/Troubleshooting.md)
3. Verify you're in the correct directory
4. Re-run `initial-bootstrap.sh` to complete setup

---

## 🔄 Regular Verification

Run verification periodically to ensure everything stays working:

```bash
# Quick check (takes <10 seconds)
./scripts/verify-setup.sh

# Full diagnostic check
./scripts/verify-setup.sh --verbose
```

---

## 📝 Understanding the Output

### ✅ PASS
Check succeeded - no action needed

### ❌ FAIL
Check failed - review the error message and troubleshoot

### ⚠️ Warning
Check passed but noted a potential issue:
- GPU not found → CPU mode will be slow but functional
- HuggingFace token not found → Some models may need authentication

---

## 🛠️ Manual Verification

If scripts don't work, manually check:

```bash
# Is Python environment there?
ls -la ~/torch-env

# Is PyTorch installed?
source ~/torch-env/bin/activate
python -c "import torch; print(torch.__version__)"

# Is vLLM installed?
python -c "import vllm; print(vllm.__version__)"

# Are helper scripts generated?
cd ~/.config/llm-doctrine
ls -la daily-bootstrap.sh test-connection.sh
```

## 💡 Key Concepts

### What is Verified vs. Not Verified

**✅ Verified (Setup Only)**
- Python environment is installed
- Dependencies are present
- Scripts are created and executable
- Configuration files exist

**❌ Not Verified (Runtime Only)**
- Can you actually launch a model? (requires GPU/time)
- Can you chat with the model? (requires model to run)
- Can you integrate with your IDE? (requires IDE setup)

To verify these, see:
